Primer Componente

Un componente es una pieza de intefaz grafica (UI) que contiene su propia funcionalidad, de tal forma es un pedazo dentro de una aplicacion. En otra palabras son funciones de Javascrips que devuelve markup (marcado) a la pagina de donde se importe. para utilizar React hay que tener instalado los Snippets de visual estudio, por ejemplo para crear un funcion componente se escribe ffc 
        
          function PrimerComponente() {
            return ( <h1>Hola mundo</h1> );
          }
        
          export default PrimerComponente;
para llamar a nuestro PrimerComponente tenemos que importarlo desde nuestro archivo main.jsx, quedando de la siguiente forma: 
      
        import React from 'react'
        import ReactDOM from 'react-dom/client'
        import ComponentePrueba from './ComponentePrueba'

        ReactDOM.createRoot(document.getElementById('root')).render(
          <React.StrictMode>
            <ComponentePrueba />
          </React.StrictMode>,
        )
con el atajo de teclado alt + Shift + F (Formatea el docuemento) en visual studio Code.
La forma recomendada en la actualidad es usar el Snippets rafc que crea una funcion flecha 
      
        import React from 'react'

        export const Componente2 = () => {
          return (
            <div>Componente2</div>
          )
        }
Pero cambia la forma de llamardo ahora en el momento de importarlo hay que colocar entre llaves { } quedando el codigo de la siguiente forma: 
      
        import { Componente2 } from './Componente2'

Variables

Nota: Los nombres de los componente debe esta con mayuscula inicial o sale error al momento de importarlo, ademas es una convencion de buenas practicas. para pasar los variables desde un componente al main.jsx para que lo renderise React, hay que escribir la variables entre llaves { }, quedando de la siguiente forma: 
      
        const wordSerntence='frase de prueba'
        const numero=145678
        const tecnologias=['React','Spring boot','node']
        const verdadero=true
        const funcionFlecha=()=>1+1
        const objeto={nombre:'Curso de javascrip',duracion:4}
        const fecha=new Date();
  
  
        export const Variables=()=>{
          return (
            <h1>{wordSerntence}</h1>
          )
        }
los campos primitivos como string, numeros, boleanos y array se pasan sin ningun problemas, las funciones deben colocar los parentesis para que se ejecute, ejemplo: 
      
        const funcionFlecha=()=>1+1
 
  
        export const Variables=()=>{
          return (
            <h1>{funcionFlecha()}</h1>
          )
        }
En cuanto a los objetos , los objeto tipo fecha y los JSON sale error para poder enviarlos hay que converirlos primeros a string usando stringify ejemplo: 
      
        const objeto={nombre:'Curso de javascrip',duracion:4}
        const fecha=new Date();

        export const Variables=()=>{
          return (
            <h1>{JSON.stringify(objeto)}</h1>
          )
        }

renderizado condicional

Como su nombre lo indica cuando se cumple una condicion se debe mostrar el componente, por ejemplo en App.jsx vamos a crear un estado showChild, y un boton de tal forma que cuando se haga click en el boton cambia el estado al valor contrario es decir, si se esta activo se desactiva y vicebersa. la condicion debe ponerse entre { } se evalua la codicion y se coloca &&, quedando la condicion asi: 
      {showChild && <CicloVida> </CicloVida>}
ahora dentro del boton con el evento onClick se coloca entre llaves un callback donde se llama cambia el estado 
      
        import { useState } from "react";
        import { CicloVida } from "./src/components/CicloVida";
        const App = () => {
          const [showChild, setShowChild]= useState();
        
          return (
            <>
              <h1>Ciclo de vida uso de usEfect()</h1>
              <button onClick={ ()=> setShowChild(!showChild) }>Llamar hijo</button>
              {showChild && <CicloVida> </CicloVida>}
           
            </>
          );
        };
        
        export default App;
Ahora vamos a crear un contador el el compoenete ciclo de vida y cada de vez que se se mueste se incremente el contador. quedando el codigo asi: 
      import { useEffect, useState } from "react"
      
      export const CicloVida = () => {
          const [contador, setContador]=useState(0)
          useEffect(()=>{
              console.log("mostrar componente renderizado")
              }
      
          )
      
        return (
          <div className="bordeCuadrado">
          CicloVida
          <h3>{contador}
          <button onClick={()= > setContador(contador+1)}>Incrementar</button>
          </div>
        )
      }
El detalle es que cada vez que se le da click al boton del componente ciclo de vida se recarga todo el componente, para que este no suceda en useEffect() se debe colocar un [ ] corchetes en blanco 
      useEffect(()=>{
        console.log("mostrar componente renderizado")
        },[]

      )
por ultimo si queremos que cuando se destrulla un componente se ejecute algo debemos retornar un return dentro del useEfect() con otro callback con el mensaje 
        useEffect(()=>{
        return()=>{

context

¿Qué es useContext? Es un hook que te permite compartir datos globales entre componentes sin tener que pasarlos por props manualmente (prop drilling).
se puede usar cuando necesitamo Compartir datos globales como el idioma, el tema (oscuro/claro), el usuario autenticado, etc.
Creamos el contexto AuthContext.js 
      import { createContext, useContext } from 'react';
      
      // Simulación del usuario autenticado
      const fakeUser = {
        name: 'Rodney',
        role: 'admin', // Cambia a 'user' o 'guest' para probar
      };
      
      export const AuthContext = createContext();
      
      export function AuthProvider({ children }) {
        return (
          <AuthContext.Provider value={{ user: fakeUser }}>
            {children}
          </AuthContext.Provider>
        );
      }
      
      export function useAuth() {
        return useContext(AuthContext);
      }
Componente para proteger rutas – PrivateRoute.js 
      
      import { Navigate } from 'react-router-dom';
      import { useAuth } from './AuthContext';
      
      export function PrivateRoute({ children, requiredRole }) {
        const { user } = useAuth();
      
        // Si el rol del usuario no coincide, redirige
        if (user.role !== requiredRole) {
          return <Navigate to="/not-authorized" replace />;
        }
      
        return children;
      }
Definir rutas protegidas App.js 
      
      import { BrowserRouter as Router, Routes, Route } from 'react-router-dom';
      import { AuthProvider } from './AuthContext';
      import { PrivateRoute } from './PrivateRoute';
      
      import HomePage from './pages/HomePage';
      import AdminPage from './pages/AdminPage';
      import NotAuthorized from './pages/NotAuthorized';
      
      function App() {
        return (
          <AuthProvider>
            <Router>
              <Routes>
                <Route path="/" element={<HomePage />} />
                
                <Route
                  path="/admin"
                  element={
                    <PrivateRoute requiredRole="admin">
                      <AdminPage />
                    </PrivateRoute>
                  }
                />
      
                <Route path="/not-authorized" element={} />
              </Routes>
            </Router>
          </AuthProvider>
        );
      }
      
      export default App;

Configurar puerto de sqlserver

primero queremos saber que puerto esta utilizando sqlserver, lo primero que debemos averiguar es el numero del servicio 
      tasklist | find /i "sqlservr.exe"
el servicio tiene el numero 23000, ahora debemos averiguar el puerto en el que corre el proceso. 
      netstat -oan | find /i "23000"
Para cambiar el puerto en el que esta corriendo el sqlserver hay que realizar: 

      para cambiar el puerto de sqlserver hay que entra a panel de control
      Herramientas del Sistemas --- Administración de Equipo
      Debajo de Administración de Disco hay uno que se llama 
      Servicios y aplicaciones
      después se escoge SQL Server Configuration Manager
      y se escoge la opción SQL server network configuración
      se abre una ventana con 2 pestañas (protocolo , IP adresss)
      hay escoger la de IP apress y buscar al final la que dice IPAII
      TCP Dynamics Port: 51537 colocamos el puerto que queremos
Al final hay que reiniciar el servicio, a veces incluso hay que reinicar el computador para que tome los cambios

Habilitar CORS

para habilitar los CORS lo realizamos en em main.ts quedando el codigo completo: 
      
      import { NestFactory } from '@nestjs/core';
      import { AppModule } from './app.module';
      import { DocumentBuilder, SwaggerModule } from '@nestjs/swagger';
      import { TrimPipe } from './common/pipes/trim.pipe.ts/trim.pipe.ts.pipe';
      
      async function bootstrap() {
        const app = await NestFactory.create(AppModule);
      
        // ✅ Habilitar CORS
        app.enableCors({
          origin: '*', // ⚠️ Cambia esto por tu dominio en producción, ej: 'http://localhost:4200'
          methods: 'GET,HEAD,PUT,PATCH,POST,DELETE',
          credentials: true,
        });
      
        // Pipe global
        app.useGlobalPipes(new TrimPipe());
      
        // swagger
        const config = new DocumentBuilder()
          .setTitle('API DIAN')
          .setDescription('CRUD Departamentos y Municipios')
          .setVersion('1.0')
          .addBearerAuth() // Habilita el botón de JWT
          .build();
      
        const document = SwaggerModule.createDocument(app, config);
        SwaggerModule.setup('api', app, document);
      
        await app.listen(3000);
      }
      bootstrap();
